Promise It Won't Hurt
10/19/2015

Node.js achieves its performance through the use of asynchronous I/O. Although this is terrific for performance, asynchronous code can be very difficult for programmers to manage. As a result, developers often have to write a lot of code that obscures the intent and spend valuable time debugging obscure problems.

JavaScript is a great language to use when dealing with asynchronicity. With the recent addition of promises, that story gets even better, because they make asynchronous code composable. Work through this workshopper to find out how.
